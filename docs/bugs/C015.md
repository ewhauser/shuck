## Delta analysis

### Safe `sudo` wrappers are over-flagged `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 14 locations / 7 scripts

The current matcher warns on any simple command named `sudo` that has redirects, even when the redirect belongs to a safe follow-up command or the command is already using the privileged helper pattern ShellCheck expects. In the sample, this showed up on `sudo -u` wrappers, `sudo tee` pipelines, and a few plain `sudo` command invocations that do not actually suffer from the redirection-order problem the rule is meant to catch.

**Samples:**
- `HariSekhon__DevOps-Bash-tools__.bash.d__mac.sh:113` - `sudo scutil <<< "..."` is flagged even though it is not a write-redirection problem.
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__audiosettings.sh:335` - `sudo -u "$__user" ... 2>/dev/null` is a helper wrapper, not a redirection-order bug.
- `asdf-vm__asdf__scripts__install_dependencies.bash:34` - `curl ... | sudo tee /etc/apt/trusted.gpg.d/microsoft.asc >/dev/null` is already using the safe `tee` handoff.
- `basecamp__omarchy__bin__omarchy-setup-dns:34` - `sudo` is used as part of a shell helper sequence, not a direct redirected write.
- `basecamp__omarchy__migrations__1754984734.sh:15` - `echo ... | sudo tee /etc/docker/daemon.json >/dev/null` is the intended privileged-write pattern.
- `termux__termux-packages__scripts__run-docker.sh:181` - `echo 65535 | sudo tee /proc/sys/kernel/pid_max >/dev/null` is another safe `tee` pipeline.
- `void-linux__void-packages__srcpkgs__nvi__patches__nvi-08-lfs.patch` - parse noise; not a useful rule signal, but it appeared in the same sample run.

### Multi-redirect `sudo tee` commands collapse into one warning `[location-only]`

**Bucket:** Location-only
**Impact:** 2 commands / 1 script

In `skyscraper.sh`, ShellCheck points at each redirected destination inside the `sudo tee` command, while Shuck reports a single broader warning span for the whole command. The underlying issue is the same, but the span anchoring and per-redirect counting do not line up yet.

**Samples:**
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__skyscraper.sh:494` - `sudo tee "$1" >/dev/null`
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__skyscraper.sh:497` - `sudo tee "$1" >/dev/null`
