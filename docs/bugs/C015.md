## Delta Analysis

### Redirection span anchoring drift `[location-only]`

**Bucket:** Location-only
**Impact:** 14 scripts in the sample

The sampled diffs are overwhelmingly not about whether `C015` fires, but about where it points. ShellCheck anchors SC2024 on the redirect text itself, while Shuck reports the wider `sudo ... > file` command span. That makes the diagnostics look different even when both tools agree that the pattern is wrong.

Representative examples:
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__audiosettings.sh:335` - ShellCheck points at the redirect region; Shuck points at the whole `sudo` command.
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__runcommand__runcommand.sh:1143` - same shape on a multi-line `sudo` command.
- `basecamp__omarchy__bin__omarchy-setup-dns:34` - repeated redirection warnings with the current span covering the full command.
- `fideloper__Vaprobash__scripts__mariadb.sh:21` - two adjacent `sudo` write sites show the same anchor drift.
- `termux__termux-packages__packages__openjdk-17__build.sh` - multi-line command chains widen the current span even further.

One script, `RetroPie__RetroPie-Setup__scriptmodules__supplementary__skyscraper.sh`, shows the same underlying issue in a slightly noisier form: ShellCheck emits four redirect-span diagnostics while Shuck emits two wider command-span diagnostics.
