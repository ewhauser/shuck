## Delta Analysis

### Literal test guards in generated scripts are over-reported `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 2 locations / 2 scripts in the sample

The sampled mismatches are both fixed-literal truthy tests, but they appear in very different script shapes.

One is a straight `[` test on a fixed string in a SlackBuild install hook:
`[ "lsmod | grep v4l2loopback_dc" ]`. Shuck flags it as a constant truthy test, but ShellCheck stays silent in the sample.

The other is a nested `if [[ ... ]]` inside a command-substitution branch in a Termux script. The warning is still coming from a literal-test shape, but the surrounding code is generated-error-message plumbing rather than a standalone runtime predicate, which makes the current matcher feel too eager in this corpus slice.

**Samples:**
- `SlackBuildsOrg__slackbuilds__multimedia__droidcam__doinst.sh:22` - fixed-string `[` test on `"lsmod | grep v4l2loopback_dc"`
