## Delta Analysis

### Unquoted array expansions in command arguments are under-reported `[shuck-fix]`

**Bucket:** ShellCheck-only  
**Impact:** 26 locations / 23 scripts in the sample

The sampled misses are almost all ordinary unquoted array expansions in command arguments. ShellCheck reports them as SC2068, but Shuck stays silent, which means `unquoted_array_expansion.rs` is still too narrow for the cases the corpus is actually exercising. The common shapes are `"$@"`-style array forwarding, `rsync ${RSYNC_OPTIONS[*]} ...`, and helper wrappers that pass array values directly into commands.

Representative examples:
- `SlackBuildsOrg__slackbuilds__academic__nootka__nootka:21` - unquoted array argument passed to a command wrapper.
- `SlackBuildsOrg__slackbuilds__games__mupen64plus-nightly__mupen64plus-nightly.SlackBuild:125-128` - multiple unquoted array arguments in one helper script.
- `termux__termux-packages__packages__tsocks__03_fixloop.patch` - parser noise around the sample tail, but the same array-forwarding shape shows up in nearby shell files.
- `void-linux__void-packages__srcpkgs__smali__files__baksmali:2` - unquoted array expansion in a packaging helper.
- `233boy__v2ray__src__log.sh:11` - ShellCheck reports a direct unquoted array expansion that Shuck misses entirely.

### Proxmox copy-data helpers show span drift on the same `rsync` pattern `[location-only]`

**Bucket:** Location-only  
**Impact:** 2 locations / 2 scripts

Two Proxmox helper scripts show the same `rsync ${RSYNC_OPTIONS[*]}` pattern, but ShellCheck and Shuck land on different invocations rather than the same line. In both files, one tool flags the earlier `rsync` call and the other flags the later one, which suggests the matcher is close but not anchored consistently across repeated helper blocks.

**Samples:**
- `community-scripts__ProxmoxVE__tools__copy-data__home-assistant-core-copy-data-home-assistant-container.sh` - ShellCheck flags the earlier `rsync` call, while Shuck flags the later one.
- `tteck__Proxmox__misc__copy-data__z2m-copy-data-z2m.sh` - the same split appears in the second Proxmox copy-data helper.
