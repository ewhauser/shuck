## Delta analysis

### Fixed literal string tests are still flagged `[shuck-correct]`

**Bucket:** Shuck-only
**Impact:** 3 locations / 3 scripts

All three sampled mismatches have the same shape: a unary string test compares a literal token instead of a runtime value, and Shuck warns while ShellCheck stays silent. The scripts use the test as part of `nvm` color-state setup, but the specific operand is still a fixed string, so the result never depends on shell input.

I checked the same scripts with ShellCheck directly through the repo's nix shell and it emitted no `SC2158` diagnostics for these locations, so this is a reviewed Shuck-side divergence rather than a parity miss on ShellCheck's side.

**Samples:**
- `nvm-sh__nvm__test__fast__Set Colors__nvm_print_versions calls nvm_get_colors:14` - `if [ -n TEMP_NVM_COLORS ]; then`
- `nvm-sh__nvm__test__fast__Unit tests__nvm set_colors:10` - `if [ -n TEMP_NVM_COLORS ]; then`
- `nvm-sh__nvm__test__fast__Unit tests__nvm_get_colors:9` - `if [ -n TEMP_NVM_COLORS ]; then`
