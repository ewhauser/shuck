## Delta Analysis

### Legacy backtick spans are wider than ShellCheck `[location-only]`

**Bucket:** Location-only  
**Impact:** 202 locations / 13 scripts in the sample

The dominant gap is not whether `S005` fires, but where it points. ShellCheck anchors SC2006 on the legacy backtick substring itself, while Shuck reports the whole word or command span that contains it. That makes the diagnostics look different even when both tools agree the backticks are the problem.

Representative examples:
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__gamecondriver.sh:46` - ShellCheck points at `` `pwd` ``, while Shuck reports the full `ln -sfn ...` word.
- `SlackBuildsOrg__slackbuilds__network__hylafax+__doinst.sh:3` and `:8` - multiple backticks inside one assignment are each anchored more narrowly by ShellCheck.
- `SlackBuildsOrg__slackbuilds__network__openresty__rc.openresty:95` - ShellCheck highlights the command substitution fragment; Shuck spans the full quoted word.
- `SlackBuildsOrg__slackbuilds__office__gnucash__gnucash.SlackBuild:143` - same anchor drift on a backtick inside a larger shell expression.
- `void-linux__void-packages__srcpkgs__drivestrike__files__drivestrike__run:4` - Shuck reports the whole echoed string, while ShellCheck highlights just the backtick region.

### Harness and wrapper files still over-report `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 126 locations / 25 scripts in the sample

The remaining tail is concentrated in harness-style and wrapper-style files. Some of the sampled cases have explicit `shellcheck disable=SC2006` comments that ShellCheck respects but Shuck still reports, which points to a suppression-path bug. Others are generated test helpers or project wrappers where ShellCheck stays silent in this sample and Shuck still flags the backticks.

Representative examples:
- `kward__shunit2__shunit2_misc_test.sh:182-184` - the file has `# shellcheck disable=SC2006`, but Shuck still reports the backticked helper calls.
- `kward__shunit2__shunit2:42` and `:48` - backticks in test-framework setup code.
- `tj__git-extras__bin__git-psykorebase:80` - a quoted backtick pair in an interactive helper message.
- `tteck__Proxmox__ct__audiobookshelf.sh:11:1-17:1` - a sourced-wrapper project file that ShellCheck does not flag in the sample.
- `tteck__Proxmox__misc__container-restore-from-backup.sh:33:1-39:1` - another wrapper-style sample with a wide Shuck-only region.
