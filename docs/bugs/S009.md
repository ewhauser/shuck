## Delta Analysis

### Echoed substitution inside nested command wrappers is under-reported `[shuck-fix]`

**Bucket:** ShellCheck-only  
**Impact:** 62 locations / 47 scripts in the sample

The sampled misses are mostly nested `echo $(...)` shapes that appear inside assignments, loops, or command pipelines. ShellCheck still warns on these, but Shuck misses them when the echo is wrapped in another substitution or used as part of a larger command line.

Representative examples:
- `SlackBuildsOrg__slackbuilds__libraries__qt4__qt4.SlackBuild:245` - `size=$(echo $(basename ${icon}) | cut -d- -f2)`
- `google__oss-fuzz__projects__tpm2-tss__build.sh:45` - `cp -v $filename $OUT/$(echo $(basename $filename .fuzz))`
- `nvm-sh__nvm__test__fast__Unit tests__nvm set_colors:32` - `OUTPUT="$(echo $(nvm set-colors rgby 2>&1) | awk ...)"` and similar lines
- `rvm__rvm__bin__rvm-prompt:103` - `architecture="$(echo "$(ruby -v)" | sed ...)"` with ShellCheck flagging the echoed substitution

### Quoted helper wrappers and project files are over-reported `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 10 locations / 6 scripts in the sample

The over-reporting tail is concentrated in wrapper-style helper files and directive-heavy scripts. Some hits are `echo` wrappers around command substitutions that ShellCheck leaves alone in this corpus slice, while others sit in project-closure files where suppression and helper-file semantics matter.

Representative examples:
- `Bash-it__bash-it__plugins__available__colors.plugin.bash:155-202` - repeated color-helper wrappers all trip `S009`
- `bitnami__containers__bitnami__consul__1__debian-12__prebuildfs__opt__bitnami__scripts__libservice.sh:27:13-28:1` - wrapper-style helper output in a project file
- `bitnami__containers__bitnami__wordpress__6__debian-12__rootfs__opt__bitnami__scripts__libmysqlclient.sh:230:9-231:1` - another helper wrapper cluster
- `community-scripts__ProxmoxVE__misc__tools.func:2092` - project-closure wrapper with a location-only mismatch
- `moovweb__gvm__examples__native__configure:514` - project-closure helper output in a generated configure script

### Most matched hits are span-only disagreements `[location-only]`

**Bucket:** Location-only  
**Impact:** 358 locations / 292 scripts in the sample

The biggest bucket is mostly a span-anchoring problem. ShellCheck and Shuck agree that the `echo $(...)` pattern is problematic, but Shuck often reports a wider or slightly shifted span than ShellCheck. This shows up in ordinary build recipes, helper scripts, and generated wrapper code.

Representative examples:
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__skyscraper.sh:182` - ShellCheck anchors the inner substitution more tightly than Shuck
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__wifi.sh:22` - `echo "$(basename $iface)"` is the same warning, but the highlighted range differs
- `SlackBuildsOrg__slackbuilds__system__spice-vdagent__files__rc.spice-vdagent:28` - similar anchor drift on a shell helper
- `rvm__rvm__scripts__alias:96` - `echo $(...)` span differs even though both tools agree on the diagnosis

### Parse-abort fixtures are corpus noise `[environment]`

**Bucket:** Environment  
**Impact:** 11 scripts in the sample

These fixtures never reached rule comparison because Shuck hit parse errors first. They are useful for corpus health tracking, but they do not indicate an `S009` parity problem on their own.

**Samples:**
