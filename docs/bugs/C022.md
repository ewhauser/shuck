## Delta analysis

### ShellSpec parameter blocks are treated like empty tests `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 1 location / 1 script

The only rule-level mismatch in the sample comes from a ShellSpec spec file that contains a `Parameters` block. ShellCheck stays quiet, but Shuck reports `C022` on the quoted parameter lines as if they were an empty `test` expression. That points to the empty-test matcher being too eager on non-standard shell DSL content.

**Samples:**
- `ko1nksm__shellspec__spec__core__clone_spec.sh:35-36` - the `Parameters` block entries `"test"` and `"test$SHELLSPEC_LF"` are misread as an empty test expression
