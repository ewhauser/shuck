## Delta Analysis

### ShellSpec harness text is over-reported `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 1 location / 1 script in the sample

The lone parity miss comes from a shellspec test file, not from a plain shell script. The file declares a `Parameters` block and surrounding shellspec DSL, and Shuck currently flags an empty-test-shaped fragment inside that harness while ShellCheck stays silent. That makes this look like a scope guard issue around non-shell harness syntax rather than a general matcher problem.

**Samples:**
- `ko1nksm__shellspec__spec__core__clone_spec.sh:35-36` - shellspec `Parameters` content is being treated as an empty test site

### Parse-abort fixtures are corpus noise `[environment]`

**Bucket:** Environment  
**Impact:** 11 scripts in the sample

These scripts never reached rule comparison because Shuck rejected them first. They are useful for overall corpus health, but they do not point to a C022-specific mismatch.
