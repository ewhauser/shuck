## Delta Analysis

### Nested `printf` calls are missed inside command substitutions `[shuck-fix]`

**Bucket:** ShellCheck-only  
**Impact:** 39 locations / 39 scripts in the sample

The sampled ShellCheck-only hits are overwhelmingly `printf` calls that live inside command substitutions or other nested command contexts. Shuck's current walker in `printf_format_variable.rs` only inspects the top-level command list, so nested `printf` nodes are invisible unless the surrounding word traversal descends into the substitution. That explains why ShellCheck sees the format-variable violation and Shuck stays silent.

Representative examples:
- `SlackBuildsOrg__slackbuilds__development__ICTester__ICTester.SlackBuild:26` - `SRCNAM="$(printf $PRGNAM | cut -d- -f2-)"`
- `SlackBuildsOrg__slackbuilds__libraries__VitaMTP__VitaMTP.SlackBuild:37` - `SRCNAM="$(printf $PRGNAM | tr '[:upper:]' '[:lower:]')"`
- `SlackBuildsOrg__slackbuilds__libraries__dropbox-python__dropbox-python.SlackBuild:28` - `SCRNAM=$(printf $PRGNAM | cut -c1-7)`
- `termux__termux-packages__packages__angle-grinder__build.sh:19` - nested `printf` used to derive `env_host`
- `SlackBuildsOrg__slackbuilds__perl__perl-Math-Int128__perl-Math-Int128.SlackBuild:39` - same nested `printf` shape in a packaging script

### Literal interpolations are being treated like variable-supplied format strings `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 1 location / 1 script in the sample

One sampled false positive comes from a literal `printf` format string that embeds a shell expansion inside the string itself. ShellCheck leaves this alone, but Shuck reports it because the rule currently treats any non-plain literal format text as a variable-supplied format string.

**Sample:**
- `paulirish__dotfiles__bin__git-chromecheckout:26` - `printf "%bChecking out: %b$url %b...%b\n" ...`

The remaining shuck-only hits are concentrated in helper-heavy or directive-heavy files. One of them explicitly has `# shellcheck disable=SC2059` and still reports under Shuck, while the others are in project-closure or unknown-shell-collapse contexts where the sample labels already suggest corpus-policy noise rather than a clean rule-local mismatch.
