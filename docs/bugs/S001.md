## Delta analysis

### Rule dispatch gap `[implementation-gap]`

**Bucket:** ShellCheck-only
**Impact:** 40,176 locations / 1,569 scripts

`S001` is registered in the rule catalog, but there is no rule module wired into `Checker::check_commands()` yet. As a result, Shuck emits no `S001` diagnostics at all, while ShellCheck reports the usual unquoted-argument cases throughout the corpus.

This is not a subtle parity mismatch. It is a missing implementation path.

**Samples:**
- `233boy__v2ray__src__log.sh:12` - multiple unquoted variable uses in command arguments.
- `CISOfy__lynis__include__report:101` - a long run of unquoted parameter expansions in ordinary argument positions.
- `RetroPie__RetroPie-Setup__scriptmodules__emulators__jzintv.sh:39` - an unquoted expansion in a shell command argument.
- `void-linux__void-packages__srcpkgs__prometheus__files__prometheus__run:6` - an unquoted variable in a service wrapper command.
