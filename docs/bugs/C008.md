## Delta analysis

### Trap span anchoring `[location-only]`

**Bucket:** Location-only
**Impact:** 15 locations / 12 scripts

Most of the sampled differences are the same shape: ShellCheck points at the specific expansion inside the trap body, while Shuck reports the whole quoted handler span. The warning is still about the same trap, but the anchor is broader than ShellCheck's and makes the diff look like a different location.

**Samples:**
- `community-scripts__ProxmoxVE__misc__tools.func:1104` - `trap "rm -rf '$dir'" EXIT`
- `docker-library__official-images__test__tests__mysql-log-bin__run.sh:18` - `trap "docker rm -vf $cid > /dev/null" EXIT`
- `docker-library__official-images__test__tests__postgres-initdb__run.sh:27` - `trap "docker rm -vf $cid > /dev/null" EXIT`
- `docker-library__official-images__test__tests__ruby-bundler__container.sh:5` - `trap "rm -rf '$dir'" EXIT`
- `dokku__dokku__plugins__scheduler-docker-local__scheduler-deploy:49` - `trap "..."` around the scheduler deploy hook

### Multi-expansion trap bodies are collapsed `[shuck-fix]`

**Bucket:** ShellCheck-only
**Impact:** 2 locations / 1 script

One sampled handler contains two variable expansions in the same trap string. ShellCheck reports both expansion sites, but Shuck currently treats the handler as a single boolean match and emits only one diagnostic for the whole action. That under-reports the number of affected expansions.

**Samples:**
- `docker-library__official-images__test__tests__ghost-basics__run.sh:48` - the `trap` body references both `$cid` and `$mysqlCid`, so ShellCheck emits two hits while Shuck emits one broad warning

### ShellCheck disable directives are ignored `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 3 locations / 3 scripts

The sampled corpus includes explicit `# shellcheck disable=SC2064` comments immediately before `trap` lines. ShellCheck suppresses those warnings, but Shuck still reports them. That makes the directive parser or suppression plumbing too narrow for this rule.

**Samples:**
- `HariSekhon__DevOps-Bash-tools__mysql__mysqld.sh:175` - `# shellcheck disable=SC2064` before a trap that builds a debug log handler
- `bitnami__containers__bitnami__etcd__3.6__debian-12__rootfs__opt__bitnami__scripts__libetcd.sh:539` - `# shellcheck disable=SC2064` before `trap "rm -f ${tmp_file}" RETURN`
- `gentoo__gentoo__app-antivirus__fangfrisch__files__fangfrisch-has-news.sh:73` - `# shellcheck disable=SC2064` before `trap "rm $t" EXIT`

## Checklist

- [ ] Narrow the diagnostic span to the same expansion-focused anchor ShellCheck uses, rather than the whole quoted trap string.
  - Update `trap_string_expansion.rs` to report per-expansion spans instead of `action.span`.
- [ ] Preserve multi-expansion traps as multiple diagnostics.
  - Walk the trap body for every variable expansion and emit one `C008` per offending expansion.
- [ ] Honor `# shellcheck disable=SC2064` for this rule.
  - Extend suppression handling so the large-corpus harness can filter these reviewed directives the same way ShellCheck does.
- [ ] Re-run the large-corpus comparison and refresh this report.
