## Delta Analysis

### Variable-Built Case Patterns `[shuck-correct]`

**Bucket:** Shuck-only  
**Impact:** 111 locations / 31 scripts

The sampled scripts consistently build `case` patterns from variables, and shuck flags those patterns while ShellCheck stays silent in this sample. The representative cases are all the same shape: a `case` arm uses a variable, a variable-derived glob fragment, or a variable plus surrounding literal text as the pattern. I treated this as reviewed shuck-side strictness rather than a parity bug because the rule is specifically about dynamic case patterns and the warnings line up with that intent.

**Samples:**
- `asdf-vm__asdf__asdf.sh:91-121` - `case ":$PATH:" in ... *":${_asdf_bin}:"* ...`
- `bittorf__kalua__openwrt-addons__etc__kalua__db:959` - `case` arm keyed by a variable-derived pattern
- `rvm__rvm__bin__rvm-prompt:61` - `case "${GEM_HOME:-""}" in (*${rvm_gemset_separator:-"@"}*)`
- `openrc__openrc__sh__rc-functions.sh:61,139` - variable-built dispatch patterns in helper logic
- `scop__bash-completion__completions-core__make.bash:82-97` - completion patterns assembled from variables
- `community-scripts__ProxmoxVE__misc__tools.func:2953-3691` - repeated variable-based case dispatch in helper functions
