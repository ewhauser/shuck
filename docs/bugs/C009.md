## Delta analysis

### Quoted literal matches are over-flagged `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 5 locations / 4 scripts

The common pattern is a quoted right-hand side whose contents are already a fixed literal string. ShellCheck stays quiet in these cases, but Shuck still emits `C009`. That suggests the current matcher is too broad and is treating every quoted regex operand as a mistake, even when the literal text does not depend on regex semantics.

**Samples:**
- `HariSekhon__DevOps-Bash-tools__packages__brew_upgrade_packages_if_outdated.sh:53` - `[[ "$output" =~ "Error: No available formula" ]]`
- `bitnami__containers__bitnami__etcd__3.6__debian-12__rootfs__opt__bitnami__scripts__libetcd.sh:90` - `[[ "$var" =~ "ETCD_CFG_CLIENT_" ]]`
- `bitnami__containers__bitnami__etcd__3.6__debian-12__rootfs__opt__bitnami__scripts__libetcd.sh:93` - `[[ "$var" =~ "ETCD_CFG_PEER_" ]]`
- `docker-mailserver__docker-mailserver__target__scripts__helpers__utils.sh:28` - `[[ $(file "${1}") =~ 'CRLF' ]]`
- `termux__termux-packages__scripts__bin__update-packages:358` - `[[ "${_CACHED_ISSUES[$number]}" =~ "'Auto update failing for ${pkg_name}'" ]]`

### Suppression directives are ignored `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 1 location / 1 script

One fixture explicitly disables SC2076 on the line immediately before the regex check. ShellCheck honors that directive, but Shuck still reports the warning, so the suppression path for this rule is incomplete.

**Samples:**
- `termux__termux-packages__scripts__bin__update-packages:357` - `# shellcheck disable=SC2076` before the quoted literal check above

### True positive baseline

**Bucket:** None
**Impact:** 1 matching location in the sample

One sampled site still matches ShellCheck cleanly and shows the rule is not entirely too strict. In this case, both tools agree that the quoted right-hand side changes the `=~` behavior.

**Samples:**
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__runcommand__runcommand.sh:263` - `[[ "$4" =~ "\\/\\+Start\\ (.+)\\.sh$" ]]`
