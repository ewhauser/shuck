## Delta Analysis

### Quoted literal RHS values are over-reported `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 5 locations / 4 scripts in the sample

`C009` currently flags any quoted `[[ ... =~ ... ]]` right-hand side, but ShellCheck only warns for the forms it considers genuinely regex-like. In the sample, fixed string comparisons such as `'CRLF'`, `"--appendconfig"`, and `"Error: No available formula"` were reported by Shuck even though ShellCheck stayed silent. The `bitnami` sample also shows two literal regex branches that ShellCheck leaves alone while Shuck warns on both.

**Samples:**
- `HariSekhon__DevOps-Bash-tools__packages__brew_upgrade_packages_if_outdated.sh:53` - quoted literal branch inside `[[ ... =~ ... ]]`
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__runcommand__runcommand.sh:1121` - quoted literal `"--appendconfig"` comparison
- `bitnami__containers__bitnami__etcd__3.6__debian-12__rootfs__opt__bitnami__scripts__libetcd.sh:90` - quoted literal `ETCD_CFG_CLIENT_`
- `bitnami__containers__bitnami__etcd__3.6__debian-12__rootfs__opt__bitnami__scripts__libetcd.sh:93` - quoted literal `ETCD_CFG_PEER_`
- `docker-mailserver__docker-mailserver__target__scripts__helpers__utils.sh:28` - quoted literal `'CRLF'`

### ShellCheck suppressions are not honored consistently `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 1 location / 1 script in the sample

The Termux sample includes an explicit `# shellcheck disable=SC2076` comment immediately before a quoted-regex comparison. ShellCheck suppresses the warning, but Shuck still reports `C009`, which means the rule is not fully respecting the suppression path for this code.

**Samples:**
- `termux__termux-packages__scripts__bin__update-packages:357-358` - `# shellcheck disable=SC2076` followed by a quoted `=~` comparison
