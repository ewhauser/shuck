## Delta analysis

### Pipeline span mismatch `[location-only]`

**Bucket:** Location-only
**Impact:** 146 scripts in the sample

The dominant difference is not whether C007 fires, but where it points. ShellCheck anchors SC2038 on the `find` command itself, while shuck reports the whole pipeline span because `find_output_to_xargs.rs` currently reports `command.span` for the entire `find ... | xargs ...` pipeline.

Representative examples:
- `SlackBuildsOrg__slackbuilds__academic__getfem__getfem.SlackBuild:105`
  - ShellCheck points at `find $PKG -iname "*.so"`
  - shuck points at the full `find ... | xargs chmod +x` pipeline
- `SlackBuildsOrg__slackbuilds__development__gprbuild__gprbuild.SlackBuild:114`
  - two adjacent `find -name ... | xargs rm` cases show the same anchoring gap
- `termux__termux-packages__packages__openjdk-17__build.sh:107`
  - multi-line `find ... | xargs ... | xargs echo` chains widen the span even further

This looks like a pure span-anchoring issue, not a semantic mismatch in the matcher.

### Directive / project-closure outlier `[shuck-only]`

**Bucket:** Shuck-only
**Impact:** 1 script / 3 locations

`bitnami__containers__bitnami__jenkins__2__debian-12__rootfs__opt__bitnami__scripts__libjenkins.sh` reports three C007 hits in shuck but none in ShellCheck. The block is labeled with `directive-handling` and `project-closure`, which suggests the delta is driven by shellcheck suppression / sourced-file behavior rather than the core `find | xargs` matcher.

Representative lines:
