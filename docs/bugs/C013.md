## Delta analysis

### `find` command substitutions inside `for` headers `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 2 locations / 2 scripts

Both sampled mismatches are cases where Shuck reports `C013` even though ShellCheck stays quiet. The common shape is a `for` loop whose iterable contains a command substitution starting with `find`, but the surrounding shell expression is more specific than the simple `for f in $(find ...)` pattern from the rule fixture. In both examples, the current matcher is wider than ShellCheck's actual trigger set.

**Samples:**
- `HariSekhon__DevOps-Bash-tools__checks__check_no_tabs.sh:34` - `for filename in $(find "${1:-.}" -type f | grep -Evf ... | sort); do`
- `termux__termux-packages__packages__libopenfec__build.sh:18` - `local include; for include in $(find . -type f -iname \*.h | sed 's@^\./@@'); do`
