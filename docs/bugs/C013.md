## Delta Analysis

### Pipeline-wrapped `find` substitutions are over-reported `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 2 locations / 2 scripts in the sample

`C013` currently flags any `for`-loop word whose nested command substitution starts with `find`, even when the substitution is a pipeline that filters or sorts the results first. ShellCheck stayed silent on both sample hits, which suggests SC2044 is narrower than the current matcher.

The common shape is `for name in $(find ... | grep ... | sort); do ...; done` or a very similar pipeline. In both cases the loop is iterating over the pipeline output, not directly over raw `find` output, so the current rule is too broad.

**Samples:**
- `HariSekhon__DevOps-Bash-tools__checks__check_no_tabs.sh:34` - `$(find ... | grep ... | sort)` inside a `for` loop
- `termux__termux-packages__packages__libopenfec__build.sh:18` - `$(find ... | sed ...)` inside a `for` loop
