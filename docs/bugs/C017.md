## Delta analysis

### Unary `-n` literal checks are missed `[shuck-fix]`

**Bucket:** ShellCheck-only
**Impact:** 3 locations / 3 scripts

ShellCheck reports constant unary `-n` tests when the operand is already a fixed literal string. Shuck does not currently flag these cases because the implementation only walks binary comparison forms. That leaves a real parity gap for the simplest constant tests.

**Samples:**
- `juewuy__ShellCrash__scripts__starts__fw_nftables.sh:206` - `[ -n "$$vm_ipv4" ]`
- `nvm-sh__nvm__test__fast__Set Colors__nvm_print_versions calls nvm_get_colors:14` - `[ -n TEMP_NVM_COLORS ]`
- `nvm-sh__nvm__test__fast__Unit tests__nvm_get_colors:9` - `[ -n TEMP_NVM_COLORS ]`

### Runtime-sensitive operators are over-matched `[shuck-fix]`

**Bucket:** Shuck-only
**Impact:** 2 locations / 2 scripts

The current matcher treats any static-looking operands in a comparison as constant, but some operators are still runtime-sensitive. The sampled `-ge` case is an arithmetic comparison where the bare word is interpreted in arithmetic context, and the `-ot` case compares file timestamps rather than fixed string values. ShellCheck stays silent on both, so the rule is too broad today.

**Samples:**
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__skyscraper.sh:685` - `[[ i -ge 10 ]]`
- `bittorf__kalua__openwrt-addons__usr__sbin__cron.webcam:338` - `[ "/webcam/lastsnap.jpg" -ot "/webcam/MARK" ]`
