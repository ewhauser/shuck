# C005: Close the SC2016 parity gap

## Summary

C005 (`single-quoted literal`) currently does not match ShellCheck's SC2016 behavior closely enough.

Observed delta snapshot:

- ShellCheck-only: 364 locations / 280 scripts
- Shuck-only: 1,523 locations / 1,006 scripts
- Location-only: 7 scripts

The current implementation in `crates/shuck-linter/src/rules/correctness/single_quoted_literal.rs` is a broad "single-quoted text containing `$...`" matcher. ShellCheck's SC2016 is narrower and includes command- and context-specific exemptions.

## What We Verified

### Location-only

These currently look like span anchoring differences, not semantic disagreements:

- `Nyr__openvpn-install__openvpn-install.sh:68`
  - ShellCheck: `68:7`
  - Shuck: `68:8`
- `basecamp__omarchy__migrations__1755870033.sh:4`
  - ShellCheck: `4:10`
  - Shuck: `4:11`

In both cases, ShellCheck points at the opening single quote and Shuck points at the first character inside the quote.

### Shuck-only false positives

Representative corpus cases:

- `HariSekhon__DevOps-Bash-tools__.bash.d__teamcity.sh:28`
  - `awk '{print $5}'`
- `paulirish__dotfiles__.aliases:22`
  - `alias hosts='sudo $EDITOR /etc/hosts'`
- `pyenv__pyenv__src__shobj-conf:103`
  - `'-shared -Wl,-h,$@'`
- `pyenv__pyenv__src__shobj-conf:119`
  - `'-G -dy -z text -i -h $@'`

These are cases ShellCheck intentionally exempts or does not consider SC2016 hits.

### ShellCheck-only misses

Representative corpus case:

- `pyenv__pyenv__src__shobj-conf:198`
  - `SHOBJ_ARCHFLAGS='-arch_only \`/usr/bin/arch\`'`

ShellCheck warns on backticks inside single quotes. Shuck currently misses this case.

### Synthetic SC2016 parity check

Replaying ShellCheck's own SC2016 examples showed the same shape:

- Shuck over-reports many ShellCheck-clean cases:
  - `awk`
  - `trap`
  - `eval`
  - `jq`
  - prompt assignments like `PS1=...`
  - `git filter-branch`
  - `rename`
  - special params like `$$`, `$?`, `$#`, `$@`, `$*`, `$!`, `$-`
- Shuck under-reports at least one real SC2016 case:
  - backticks inside single quotes

## Original Root Cause

The original C005 behavior was driven by two broad pieces of logic:

- It reports any single-quoted `WordPart::Literal` whose text matches `contains_expansion_like_text(...)`.
- `contains_expansion_like_text(...)` matches `$` followed by `{`, `(`, alnum, `_`, and several special parameters.

What the original implementation did not do:

- recognize backticks as SC2016 candidates
- apply ShellCheck's command-specific exemptions
- apply ShellCheck's assignment-specific exemptions
- special-case `sed`
- anchor the diagnostic span on the opening quote

Sample summary:

- Supported fixtures sampled: 3,379
- Shuck parse skips: 11
- ShellCheck execution errors: 0
- ShellCheck SC2016 hits: 510
- Shuck C005 hits: 498

Mismatch buckets for this sample:

- ShellCheck-only: 26 locations / 3 scripts
  - sample: `bittorf__kalua__openwrt-addons__etc__kalua__weblogin`
- Shuck-only: 14 locations / 2 scripts
  - sample: `kward__shunit2__shunit2_misc_test.sh`
- Location-only: 0 scripts

The location-only bucket dropped to zero in this sample after switching the reported span to the full single-quoted region (`'...'`) instead of anchoring only the start column.

Note: an earlier one-off sample run reported `73` Shuck-only locations across `14` scripts. That turned out to be inflated because the temporary comparison runner was not applying suppression directives. After rerunning with suppression handling, the true residual Shuck-only sample is `14` locations across `2` scripts.

## Remaining Delta Categories

After the suppression-aware rerun, the residual sample breaks down into a few narrow buckets.

### ShellCheck-only

- Parameter-expansion fragments inside `${var:+...}`:
  - `bittorf__kalua__openwrt-addons__etc__kalua__weblogin:189`
  - ShellCheck flags all three `'$font_color'` fragments inside `${font_color:+ ... }`
  - Shuck misses them entirely
- Single-quoted text inside backtick command substitutions:
  - `google__oss-fuzz__projects__unbound__build.sh:31`
  - ShellCheck flags the `make --eval '...'` argument
  - Shuck misses it
- Autoconf/libtool multiline quoting:
  - `moovweb__gvm__examples__native__configure`
  - Remaining misses include:
    - multiline backtick `sed -e '...'` blocks
    - large single-quoted strings broken up with `'\''`
    - one multiline single-quoted `ac_script='...'` block

### Shuck-only

- Bare numeric ShellCheck suppressions are not honored:
  - `kward__shunit2__shunit2_misc_test.sh:98`
  - `kward__shunit2__shunit2_misc_test.sh:102`
  - `# shellcheck disable=2016` is respected by ShellCheck but not currently by Shuck
- Duplicate C005 emission for adjacent mixed-quote constructs:
  - `moovweb__gvm__examples__native__configure`
  - Shuck can emit the same C005 twice for one logical mixed-quoted literal where ShellCheck emits one SC2016
