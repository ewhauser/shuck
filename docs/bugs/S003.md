## Delta Analysis

### Command-output `for` loops are over-reported `[shuck-fix]`

**Bucket:** Shuck-only  
**Impact:** 337 scripts / 388 locations in the sample

`S003` currently flags any unquoted command substitution that appears in a `for ... in ...` header. In this sample, ShellCheck stayed silent on every hit, which means the matcher is broader than the SC2045 shape the corpus is exercising. The common pattern is a `for` loop over raw command output or pipeline output, but the sample shows that not every such loop is something ShellCheck wants to warn about.

Representative examples:
- `GameServerManagers__LinuxGSM__.github__workflows__serverlist-validate.sh:28` - `for shortname in $(tail -n +2 serverlist.csv | cut -d ',' -f1); do`
- `HariSekhon__DevOps-Bash-tools__checks__check_no_tabs.sh:34` - `for filename in $(find "${1:-.}" -type f | grep -Evf ... | sort); do`
- `RetroPie__RetroPie-Setup__scriptmodules__supplementary__runcommand__runcommand.sh:733` - `for key in $(get_resolutions); do`
- `SlackBuildsOrg__slackbuilds__academic__WolframEngine__WolframEngine.SlackBuild:81` - `for i in $(find -L ... -printf "%f\n"); do`
- `rvm__rvm__scripts__snapshot:178` - `for snapshot_package in $(\command \cat "$snapshot_temp_path/pkg")`

The recurring shape is a `for` header whose command substitution is being treated as a blanket violation even when the surrounding corpus and ShellCheck do not agree.
