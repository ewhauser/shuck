## Global Verification Loop

Before checking any item complete:

1. Add a focused regression test for the broken shape in `crates/shuck-semantic/src/lib.rs` or `crates/shuck-linter/src/lib.rs`.
2. Run `cargo test -p shuck-semantic`.
3. Run `cargo test -p shuck-linter`.
4. Run `cargo test -p shuck --test large_corpus`.
5. Run `make test-large-corpus 2>&1 | tee /tmp/c001-large-corpus.log || true`.
6. Confirm the canonical example locations for the item no longer appear in the `SC2034` compatibility diff.
7. Do not add a large-corpus allowlist entry for an implementation bug. Only reviewed semantic divergences belong in `crates/shuck/tests/testdata/allowlists/`.

### ShellCheck-Only Tail

- `19` locations: name-only declarations such as `local j`, `local GENERAL`, `local cur opts base`
  - Keep as intentional divergence. This is the policy above, not a shuck bug.
- `5` locations: project-closure / dynamic-scope values that shuck intentionally keeps live
  - Examples:
    - `biosdir="$datadir/BIOS"`
    - `romdir="$datadir/roms"`
    - `emudir="$rootdir/emulators"`
    - `configdir="$rootdir/configs"`
    - `md_mode="remove"`
  - Do not “fix” unless shuck explicitly chooses to abandon project-closure and dynamic-scope semantics.
- `4` locations: SlackBuild branch-join assignments like `LIBDIRSUFFIX=""`
  - ShellCheck still warns on these in a few files even though the value is read later via `/usr/lib${LIBDIRSUFFIX}`.
  - Treat these as ShellCheck false positives, not shuck bugs.
- `10` locations: standalone env-style assignments that shuck currently suppresses too aggressively
  - Examples:
    - `CFLAGS="$SLKCFLAGS"`
    - `CFLAGS="$SLKCFLAGS" \`
    - `CXXFLAGS="$SLKCFLAGS" \`
  - These are worth fixing. The current command-prefix environment-assignment escape hatch appears too broad and is swallowing cases where no command actually follows on the same logical command.
- `5` locations: unused `for` loop variables that ShellCheck reports and shuck misses
  - Examples:
    - `for i in $(ls -1 etc/udev/rules.d/rtl-sdr.rules); do`
    - `for i in {1..10}`

- `1` script / `2` locations: pure span mismatch on array assignments
  - `PLUGINS[0]=""`
  - `PLUGIN_PARAMS[0]=""`
  - Low priority. This is not a semantic mismatch.

### Recommended Order

1. Fix branch-join / reaching-definition accounting for the SlackBuild `SLKCFLAGS` / `LIBDIRSUFFIX` family.
2. Fix `read` targets, initialized declarations, append assignments, array element assignments, and conditional command-substitution assignments.
3. Fix `for`-loop variable consistency and the remaining command-prefix environment-assignment edge cases.
4. Keep name-only declarations, project-closure globals, and the `g15daemon` span-only case out of the implementation queue unless policy changes.
